Техническое задание на разработку CRM-системы для бассейна (школьники)
1. Общие положения
1.1. Наименование системы
CRM-система для бассейна «АкваДети» (рабочее название).

1.2. Назначение системы
Автоматизация процессов записи детей в группы по плаванию, ведение расписания, учёт посещаемости, управление тренерским составом и группами, взаимодействие с родителями через интеграцию с VK.

1.3. Бизнес-цели
Снизить ручную работу администратора по обработке заявок.

Обеспечить прозрачный учёт посещаемости для родителей.

Дать тренерам удобный инструмент для работы с группами.

Организовать централизованное управление расписанием и группами.

2. Пользовательские роли
Роль	Описание	Способ авторизации
Родитель	Подаёт заявку на обучение, отслеживает успеваемость и посещаемость ребёнка	По связке телефон/e-mail или через VK ID (интеграция)
Тренер	Управляет своими группами, ведёт журнал посещаемости, составляет расписание	Логин/пароль, выдаётся администратором
Администратор	Обрабатывает заявки, управляет тренерами, группами, имеет полный доступ к данным	Логин/пароль (суперпользователь)
3. Функциональные требования
3.1. Модуль «Родитель»
3.1.1. Публичная часть (без авторизации)
Форма заявки на обучение.
Поля:

ФИО родителя (обязательное)

Телефон родителя (обязательное, валидация формата РФ)

E-mail родителя (опционально, валидация формата)

ФИО ребёнка (обязательное)

Класс (числовой, 1–11) (обязательное)

Возраст (числовой, 3–17) (обязательное)

Смена (выпадающий список: «день», «вечер») (обязательное)

Школа (текстовое поле, обязательно)

Желаемое количество занятий в неделю (выпадающий список: 1, 2, 3) (обязательное)

Год обучения плаванию (числовой, от 1 до N) (обязательное)

Отправка заявки. После успешной отправки — уведомление «Спасибо, заявка принята. Ожидайте подтверждения».

3.1.2. Личный кабинет родителя (после авторизации)
Родитель видит список всех своих детей (привязанных к его аккаунту).
Для каждого ребёнка отображается:

ФИО ребёнка

Группа (название, расписание)

ФИО тренера

Посещаемость: календарь или таблица с датами и статусами (присутствовал / отсутствовал / болел)

Год обучения

Смена

Родитель не может самостоятельно редактировать данные ребёнка. Все изменения — через модерацию администратором или тренером.

3.1.3. Авторизация через VK
Кнопка «Войти через VK» на форме входа.

Получение согласия на доступ к имени, фамилии, телефону (если есть), e-mail (если есть).

Сопоставление с существующими заявками по ФИО+телефону для автоматической привязки детей в ЛК.

3.2. Модуль «Тренер»
3.2.1. Управление группой
Тренер видит только свои группы.
Для каждой группы доступны действия:

Просмотр списка обучающихся (таблица с полями: ФИО, возраст, школа, класс, год обучения, контакты родителя).

Добавление ученика в группу вручную (администратор может делегировать это право тренеру; по умолчанию тренер может добавлять ученика, если одобрена заявка).

Удаление ученика из группы (с указанием причины: отчисление, перевод в другую группу, завершение обучения; причина сохраняется в истории).

Редактирование данных ученика (ФИО, школа, класс, год обучения, возраст, смена, количество занятий). Редактирование ограничено полями, связанными с учебным процессом (не контакты родителя).

3.2.2. Журнал посещаемости
Просмотр: календарный вид (по датам) или списочный.

Отметка присутствия/отсутствия (с подстатусами: болеет, пропуск без причины, уважительная причина).

Возможность ставить отметки массово (на всю группу за одно занятие) и индивидуально.

История посещаемости за весь период обучения.

3.2.3. Расписание
Создание и редактирование расписания для своих групп.

Параметры занятия: день недели, время начала, время окончания, место (если несколько дорожек/чаш).

Расписание может быть регулярным (шаблон на неделю) или разовым (праздничные дни, отмены).

Тренер не может менять расписание чужих групп.

3.3. Модуль «Администратор»
3.3.1. Обработка заявок
Таблица всех входящих заявок со статусами: «Новая», «В обработке», «Одобрена», «Отклонена».

Фильтрация по возрасту, году обучения, смене.

Карточка заявки содержит все заполненные поля.

Действия:

Одобрить. При одобрении администратор выбирает подходящую группу автоматически (по возрасту и году обучения) или назначает вручную. Система проверяет соответствие (если возраст/год не совпадают — предупреждение). После назначения группы ребёнок автоматически зачисляется, родителю на телефон/e-mail/VK отправляется уведомление.

Отклонить с обязательным указанием причины (текстовое поле). Родитель получает уведомление.

Если автоматический подбор группы невозможен (нет свободных мест, не подходит расписание), администратор может создать новую группу или отклонить заявку с пояснением.

3.3.2. Управление тренерами
Создание, редактирование, удаление учётных записей тренеров.

Поля тренера: ФИО, телефон, e-mail, логин, пароль, специализация, активен/неактивен.

Привязка тренера к группам (один тренер может вести несколько групп, одна группа — один тренер).

При удалении тренера группа остаётся без тренера, администратор должен назначить нового.

3.3.3. Управление группами
Создание, редактирование, удаление групп.

Параметры группы: название, возрастной диапазон (мин/макс), год обучения, максимальное количество учеников, смена (день/вечер), тренер, расписание.

При удалении группы, если в ней есть ученики, администратор обязан их перевести в другую группу или отчислить.

3.3.4. Расширенные права тренера
Администратор имеет те же права, что и тренер, но для любой группы: может редактировать состав, вести посещаемость, менять расписание.

3.4. Общее для всех ролей
Адаптивная вёрстка (мобильные устройства, планшеты, ПК).

Уведомления (внутрисистемные, e-mail, СМС-оповещения — с возможностью настройки администратором).

4. Интеграция с VK
VK ID (OAuth 2.0): авторизация родителей через ВКонтакте.

При первой авторизации создаётся учётная запись родителя с привязкой VK ID. Если телефон или e-mail совпадают с ранее указанным в заявке, происходит слияние данных и автоматическая привязка детей.

Отправка уведомлений (опционально): через ВК-бота родителю могут приходить напоминания о занятиях, подтверждение заявки, информация об отменах.

ВК-бот (отдельный контур):

Приветственное сообщение с возможностью подать заявку (ссылка на веб-форму или inline-форма).

Команда «Мои дети» — быстрый просмотр статуса группы и ближайшего занятия.

Получение push-уведомлений из системы.

5. Нефункциональные требования
5.1. Производительность
Время отклика интерфейса — не более 2 секунд при одновременной работе до 100 пользователей.

5.2. Безопасность
Пароли хранятся в хешированном виде (bcrypt).

Разграничение доступа строго по ролям (RBAC).

Защита от CSRF, XSS, SQL-инъекций.

Логирование действий администратора (кто, когда, что изменил).

5.3. Резервное копирование
Ежедневное автоматическое резервное копирование базы данных.
